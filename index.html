Complete Auth Flow Guide

Login, Register, Logout — from zero to full understanding. Every line explained.

1. What is Authentication?

Authentication is the process of verifying who you are. When you register, you create an identity. When you log in, you prove that identity. When you log out, you destroy the session.

Register Create account
→
Login Prove identity
→
Logout Destroy session

The Three States

Stateuser valueTokenScreen shown
Not logged innullNoneLogin / Register
Logged in{id, email, name}JWT token storedHome / Tabs
Logged outnullRemovedLogin

Key Concepts

Token (JWT): A string the server gives you after login. You store it and send it with every request to prove "I am who I say I am." Think of it like an ID card.
Context: A React feature that lets you share data (like user and token) with every component in your app without passing props through every level.
Persistence: Tokens survive app restarts because they're saved to SecureStore (native) or localStorage (web).

2. Project Architecture

Every file has exactly one job. Here's the complete structure:

  • app/ ← Routes (file-based routing)
    • _layout.tsx ← Root layout: wraps everything in AuthProvider
    • index.tsx ← Entry gate: checks auth, redirects to login or home
    • (auth)/ ← Group: unauthenticated screens
      • login.tsx ← Login screen
      • register.tsx ← Register screen
    • (tabs)/ ← Group: authenticated screens
      • _layout.tsx ← Tab navigator
      • index.tsx ← Home screen (first tab)
  • context/ ← Global state
    • authContext.tsx ← Creates auth state + exposes login/register/logout
  • hooks/ ← Reusable stateful logic
    • useAuth.ts ← Shortcut to consume auth context
  • services/ ← API communication
    • auth.ts ← Auth API calls (login, register, getProfile)
    • api.ts ← Generic HTTP client with token header
  • types/ ← TypeScript definitions
    • auth.ts ← User, AuthResponse, AuthContextType interfaces
  • utils/ ← Pure helper functions
    • storage.ts ← Token persistence (SecureStore / localStorage)
    • validator.ts ← Input validation (email, password)

3. Register (Sign Up) Flow

Step-by-Step Flow

Step 1
User types name, email, password in form
↓
Step 2
User taps "Register" button
↓
Step 3
handleRegister() is called
↓
Step 4
register(name, email, password) from auth context is called
↓
Step 5
Auth context calls registerUser() service
↓
Step 6
Service returns { token, user }
↓
Step 7
Context saves token via setToken() and sets user state
↓
Step 8
Screen navigates to /(tabs)

Code: Register Screen

app/(auth)/register.tsx TSX 
import { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity } from 'react-native';
import { Link, useRouter } from 'expo-router';
import { useAuth } from '../../hooks/useAuth';

export default function Register() {
    // Form state — holds what the user types
    const [name, setName] = useState('');
    const [email, setEmail] = useState('');
    const [password, setPassword] = useState('');
    const [error, setError] = useState('');

    // Get the register function from auth context
    const { register } = useAuth();

    // Get the router for navigation
    const router = useRouter();

    // Called when user taps "Register"
    const handleRegister = async () => {
        try {
            setError('');                // Clear old errors
            await register(name, email, password); // Call auth context
            // register() saves token + sets user in context
            router.replace('/(tabs)');   // Navigate to home
        } catch (err) {
            setError('Registration failed'); // Show error to user
        }
    };

    return (
        <View>
            <TextInput placeholder="Name" value={name} onChangeText={setName} />
            <TextInput placeholder="Email" value={email} onChangeText={setEmail} />
            <TextInput placeholder="Password" value={password} onChangeText={setPassword} />
            <TouchableOpacity onPress={handleRegister}>
                <Text>Register</Text>
            </TouchableOpacity>
        </View>
    );
}

Code: Auth Context Register Function

context/authContext.tsx TSX 
// This runs when the screen calls register(name, email, password)
const register = async (name: string, email: string, password: string) => {
    // Step 1: Call the service (API call)
    const response = await registerUser(name, email, password);

    // Step 2: Save the token to storage (persists across restarts)
    await setToken(response.token);

    // Step 3: Set the user in state (triggers re-render)
    setUser(response.user);
    // After this, isAuthenticated becomes true
    // Any component reading user/isAuthenticated will re-render
};

Code: Service Layer

services/auth.ts TS 
export async function registerUser(
    name: string,
    email: string,
    password: string
): Promise<AuthResponse> {
    await delay(1000); // Simulate network delay

    // In real apps, this would be a POST request to /api/register
    // fetch('/api/register', { method: 'POST', body: JSON.stringify({name, email, password}) })

    return {
        token: 'mock-jwt-token-' + Date.now(),
        user: { id: '1', email, name }
    };
}

Code: Storage Utility

utils/storage.ts TS 
export async function setToken(token: string) {
    if (isWeb) {
        // Web: use localStorage (accessible via JS)
        localStorage.setItem('authToken', token);
    } else {
        // Native: use SecureStore (encrypted, secure)
        await SecureStore.setItemAsync('authToken', token);
    }
}
Register Complete: After all 8 steps, the user is logged in, the token is stored, and they see the home screen.

Login is almost identical to register — the only difference is you verify existing credentials instead of creating new ones.

↓
Step 2
User taps "Login" button
↓
Step 3
handleLogin() is called
↓
Step 4
Auth context calls loginUser(email, password)
↓
Step 5
Service validates credentials, returns { token, user }
↓
Step 6
Context saves token + sets user
↓
Step 7
Screen navigates to /(tabs)

Code: Login Screen

app/(auth)/login.tsx TSX 
export default function Login() {
    const [email, setEmail] = useState('');
    const [password, setPassword] = useState('');
    const [error, setError] = useState('');
    const { login } = useAuth();
    const router = useRouter();

    const handleLogin = async () => {
        try {
            setError('');
            await login(email, password);  // Call auth context
            // login() saves token + sets user in context
            router.replace('/(tabs)');     // Navigate to home
        } catch {
            setError('Invalid email or password');
        }
    };

    return (
        <View>
            <TextInput placeholder="Email" value={email} onChangeText={setEmail} />
            <TextInput placeholder="Password" value={password} onChangeText={setPassword} />
            <TouchableOpacity onPress={handleLogin}>
                <Text>Login</Text>
            </TouchableOpacity>
        </View>
    );
}

Code: Auth Context Login Function

context/authContext.tsx TSX 
const login = async (email: string, password: string) => {
    // Step 1: Call the service (validates credentials)
    const response = await loginUser(email, password);

    // Step 2: Save token to storage
    await setToken(response.token);

    // Step 3: Set user in state
    setUser(response.user);
};

Code: Service with Validation

services/auth.ts TS 
export async function loginUser(
    email: string,
    password: string
): Promise<AuthResponse> {
    await delay(1000);

    // Mock validation: password must be "password"
    if (password !== 'password') {
        throw new Error('Invalid email or password');
        // This error bubbles up to handleLogin()'s catch block
    }

    return {
        token: 'mock-jwt-token-' + Date.now(),
        user: { id: '1', email, name: email.split('@')[0] }
    };
}
Key difference from register: Login validates that the user exists. If credentials are wrong, it throws an error. Register always succeeds (in this mock).

5. Logout Flow

Logout is the simplest operation — it destroys the session and sends the user back to login.

Step-by-Step Flow

Step 1
User taps "Logout" button
↓
Step 2
logout() from auth context is called
↓
Step 3
Token is removed from storage
↓
Step 4
User state is set to null
↓
Step 5
App detects user === null, redirects to login

Code: Auth Context Logout Function

context/authContext.tsx TSX 
const logout = async () => {
    // Step 1: Remove token from storage
    await removeToken();

    // Step 2: Set user to null (triggers re-render)
    setUser(null);
    // isAuthenticated automatically becomes false (derived from !!user)
    // Any screen checking user will now see null
};

Code: Storage Remove Token

utils/storage.ts TS 
export async function removeToken() {
    if (isWeb) {
        localStorage.removeItem('authToken');
    } else {
        await SecureStore.deleteItemAsync('authToken');
    }
}

Code: Usage in a Screen

app/(tabs)/index.tsx TSX 
import { useAuth } from '../../hooks/useAuth';

export default function Home() {
    const { user, logout } = useAuth();

    return (
        <View>
            <Text>Welcome, {user?.name}!</Text>
            <TouchableOpacity onPress={logout}>
                <Text>Logout</Text>
            </TouchableOpacity>
        </View>
    );
}
Why no router.replace() here? Because setting setUser(null) makes isAuthenticated false. The route guard in app/index.tsx detects this and redirects to login automatically. Logout doesn't need explicit navigation.

6. Route Guard (Entry Gate)

This is the file that decides which screen to show when the app first loads.

Code: index.tsx

app/index.tsx TSX 
import { Redirect } from 'expo-router'
import { useAuth } from '../hooks/useAuth'

export default function index() {
    const { user, isLoading } = useAuth();

    // Step 1: Show loading while checking if token exists
    if (isLoading) {
        return <ActivityIndicator />;
    }

    // Step 2: If user exists (token was valid), go to tabs
    if (user) {
        return <Redirect href="/(tabs)" />;
    }

    // Step 3: No user = not logged in, go to login
    return <Redirect href="/(auth)/login" />;
}

How It Works on App Start

App Launches
AuthProvider mounts, isLoading = true
↓
AuthProvider useEffect runs
Checks storage for saved token via getToken()
↓
Token found?
If yes: calls getProfile(), sets user. If no: user stays null.
↓
isLoading = false
index.tsx re-renders, redirects based on user value

Code: Auth Provider (App Start Logic)

context/authContext.tsx TSX 
export function AuthProvider({ children }: { children: ReactNode }) {
    const [user, setUser] = useState<User | null>(null);
    const [isLoading, setIsLoading] = useState(true);

    useEffect(() => {
        async function loadUser() {
            try {
                // Check if we have a saved token
                const token = await getToken();

                if (token) {
                    // Token exists — use it to get user data
                    const userData = await getProfile();
                    setUser(userData);  // User is restored!
                }
            } catch {
                // Token was invalid/expired — remove it
                await removeToken();
            } finally {
                setIsLoading(false);  // Done loading, allow redirect
            }
        }
        loadUser();
    }, []); // Empty array = runs once on mount

    return (
        <AuthContext.Provider value={{
            user,
            isLoading,
            isAuthenticated: !!user,  // Derived boolean
            login, register, logout
        }}>
            {children}
        </AuthContext.Provider>
    );
}
Why does index.tsx redirect, not navigate? Because <Redirect> is declarative — it tells Expo Router "always go here." It's not an imperative router.push(). Once the app passes index.tsx, the redirect doesn't fire again. That's why login/register need explicit router.replace().
